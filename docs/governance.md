## Политика логов и персональных данных

**Что логируем**
- технические события: запросы (тип/тайминг), статусы интеграций, ошибки, стоимость LLM, id артефактов
- минимум контента: без текста стратегии целиком (или хранить отдельно с контролем доступа), без сырого портфеля в логах

**Что НЕ логируем**
- токены/ключи, initData, номера счетов в явном виде, персональные идентификаторы
- любые секреты/headers

**Хранение**
- токены брокера: шифрование at-rest, доступ только backend, TTL/ротация при возможности
- снапшоты портфеля/стратегии: хранить минимум, TTL (например 7–30 дней для PoC), возможность удаления по запросу
- артефакты (PNG): без чувствительных данных на изображениях (не рисовать account_id, полные идентификаторы)

## Защита от injection и подтверждение действий

**Injection guardrails**
- инструменты выполняются только по allowlist (никаких “произвольных” tool calls)
- пользовательский ввод не может менять права доступа (нет “включи режим администратора”, “покажи токен”)
- Strategy Agent приводит ввод к слотам; дальнейшие агенты работают со слотами, а не с сырым текстом

**Text2SQL guardrails**
- LLM генерирует SQL, но выполнение — только через Data Retrieval Tool
- allowlist схемы/вьюх, запрет DDL/DML, обязательный `WHERE user_id = ...`, лимиты строк и времени

**Подтверждение действий**
- в PoC нет сделок и денежных операций
- любые чувствительные действия (подключение/отключение брокера, удаление данных, сброс стратегии) — через явное подтверждение (“Подтвердить / Отмена”)
- перед сохранением стратегии можно показывать “как система поняла стратегию (слоты)” и просить подтверждение

**Safe output**
- в ответах всегда показывать даты/источники данных (timestamp котировок, дата макро)
- если данных недостаточно — явный статус “недостаточно данных”, без догадок
